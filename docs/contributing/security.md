# security

## exploitable bugs

signalis provides no features intended explicitly for ensuring the security of developers' code. however, some classes
of bugs may have significant exploitability in practice. if it is unlikely that a bug may be exploitable in software
using signalis, it is best if you go through [bug reports](https://github.com/luau-ecs/signalis/issues/new?template=bug-report.yml) instead.

## reporting exploitable bugs

if a bug is likely or guaranteed to be present in all versions of signalis, can be predictably and exploitably triggered
by users, or is already used to attack production games, please report it at <a href="mailto:admin@nnull.co">admin@nnull.co</a>
or via [github private security reports](https://github.com/luau-ecs/signalis/security/advisories/new)
